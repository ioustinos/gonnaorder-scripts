📋 Settings Copy

GonnaOrder Scripts

Export every copyable setting from a source store to a JSON file, then apply the parts you pick to a target store — settings keys, store details, and schedules (with opening/pickup/delivery-hour assignments).

1 · Credentials

🔒 Credentials are sent over HTTPS to our serverless function, used once to get a session token, and discarded. Your email is remembered in this browser; your password is not. The same login is used against both API bases — a separate session token is minted per base.

2 · Export from source store

Store: –
Settings keys: 0
Schedules: 0
Schedule assignments: 0
Delivery zones: 0

3 · Apply to target store

The target store is read first and every difference is listed below — nothing is written until you press Apply.

4 · Pick what to copy

Store details

name & URL alias default OFF — store identity
Country and currency are locked by GonnaOrder ("contact us to change") and are never copied. All checked fields are sent in ONE update — if it fails, nothing in this group is changed.

Settings keys

Copied with merge semantics — unchecked keys on the target are untouched. Keys only the target has are never deleted.

Schedules

Schedules are created on the target (matched by name — an existing schedule with the same name is reused, never duplicated), then the opening/ordering-hour assignments are pointed at them.
